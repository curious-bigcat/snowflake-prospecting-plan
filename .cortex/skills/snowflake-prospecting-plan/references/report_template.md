## Formatting Guidelines

- Use tables for structured data comparisons
- Bold key findings and names
- Include source citations where possible (web search results)
- Mark data gaps clearly: "Limited Data — Requires Direct Discovery"
- Keep Executive Summary under 15 lines
- Prioritized opportunities should have clear scoring justification
- Engagement strategy should be actionable, not generic

---

## PDF Export Instructions

After generating the text report above, convert it to a branded PDF using Phase 8.

### Step 1: Assemble JSON Data

Convert the report content into a JSON object matching the `templates/report.html` Jinja2 variables. Key mapping:

| Report Section | JSON Key(s) |
|---|---|
| Header metadata | `company_name`, `report_date`, `prepared_for`, `industry`, `geography`, `relationship` |
| 1. Executive Summary | `executive_summary` (object with `revenue`, `employees`, `key_finding`, `top_opportunities`, `maturity`) |
| 2. Stakeholders | `stakeholder_groups` (dict of group_name → list of stakeholder objects), `org_insights` |
| 3. Financials | `financials` (object with `revenue`, `growth`, `profitability`, `segments`, `highlights`, `investments`) |
| 4. Maturity | `maturity_scores` (list of objects), `maturity_findings`, `maturity_gaps` |
| 5. Tech Landscape | `tech_landscape` (list of objects), `vendor_relationships` |
| 6. Signals | `announcements`, `analyst_coverage`, `industry_trends`, `partnerships` |
| 7. Competitive | `competitors` (list of objects), `competitive_pressures`, `differentiation` |
| 8. Risks | `risks` (list of objects with `issue`, `category`, `mitigation`) |
| 9. Initiatives | `initiatives` (list of objects with `initiative`, `status`, `outcome`, `alignment`) |
| 10. Opportunities | `opportunities` (list of objects with `use_case`, `priority`, `capabilities`, `impact`, `readiness`, `score`) |
| 11. Engagement | `engagement` (object with `entry_point`, `lead_use_case`, `poc`, `key_messages`, `objections`) |
| 12. Next Steps | `next_steps` (list of strings) |
| Sources | `sources` (list of strings) |

### Step 2: Generate PDF

```bash
# Save JSON data file
cat > {company_slug}_report_data.json << 'ENDJSON'
{ ... assembled JSON ... }
ENDJSON

# Run PDF generation
uv run --project .cortex/skills/snowflake-prospecting-plan \
  python .cortex/skills/snowflake-prospecting-plan/scripts/generate_pdf.py \
  --data {company_slug}_report_data.json \
  --output {company_slug}_prospecting_plan.pdf

# Clean up temp file
rm {company_slug}_report_data.json
```

### PDF Design Notes

The PDF template (`templates/report.html`) uses Snowflake Inc's brand design:

- **Cover page**: Navy-to-blue gradient with snowflake icon, company name, metadata
- **Section headers**: Gradient blue banners (Dark Blue → Snowflake Blue)

- **Priority badges**: HIGH (red #E53E3E), MEDIUM (orange #ED8936), LOW (green #48BB78)
- **Callout boxes**: Light blue background with blue left border
- **Score cards**: Navy gradient cards with blue accent numbers
- **Footer**: Page numbers + "Confidential — Snowflake AI & Data Cloud"
- **Page size**: A4 with proper print margins
